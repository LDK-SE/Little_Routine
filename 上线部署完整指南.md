## 第一步：注册微信小程序（提前做，审核需 1-3 天）

### 1.1 注册账号

1. 访问 [mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 右上角"立即注册" → 选择"小程序"
3. 按提示填写邮箱、密码
4. 激活邮箱（收邮件点链接）

### 1.2 选择主体类型

| 主体 | 要求 | 能否微信支付 | 认证费 |
|------|------|:----------:|:------:|
| **企业**（推荐） | 营业执照 + 法人身份证 | ✅ | 300 元/年 |
| **个体工商户** | 营业执照 + 经营者身份证 | ✅ | 300 元/年 |
| 个人 | 身份证 | ❌ | 免费 |

### 1.3 企业认证流程

1. 填写企业信息：营业执照注册号、企业名称、法人姓名
2. 填写管理员信息：姓名、身份证号、手机号、绑定微信的微信号
3. 上传材料：营业执照照片、法人身份证正反面
4. **支付认证费**：微信扫码支付 **300 元/年**
5. 管理员微信扫码确认 → 提交
6. 等待审核：通常 **1-3 个工作日**
7. 审核通过后，进入「设置 → 基本设置」复制 **AppID**，后面要用

## 第二步：购买服务器

### 2.1 为什么选腾讯云轻量应用服务器

- 与微信小程序同属腾讯生态，网络互通最好
- 自带固定公网 IP，无需额外购买
- 套餐打包流量，费用可控
- 控制台有手机版，随时管理

### 2.2 配置如何选择

| 配置 | 月费 | 适用场景 |
|------|:----:|------|
| 2核2G / 50G SSD / 3M | 68 元 | 1-3 人店，日订单 < 100 |
| **2核4G / 80G SSD / 5M** | **112 元** | **3-10 人店，日订单 100-500（推荐）** |
| 4核8G / 120G SSD / 8M | 158 元 | 连锁多店，日订单 500+ |

> 本项目需要同时跑 NestJS API + MySQL + Redis + Nginx + Dify AI，建议至少 **2核4G（112元/月）**，4核8G 更从容。

### 2.3 购买步骤

1. 打开 [cloud.tencent.com/product/lighthouse](https://cloud.tencent.com/product/lighthouse)
2. 点击"立即购买"
3. 逐项选择：
   - **地域**：选离你最近的（广州/上海/北京），不要选香港（需备案的话选大陆）
   - **镜像**：**Ubuntu 22.04 LTS**
   - **套餐**：2核4G 或 4核8G
   - **时长**：先买 **1 个月**测试，满意后年付（约 8.5 折）
4. 点击"立即购买" → 微信/支付宝扫码支付

### 2.4 服务器到手后立即做的 3 件事

**① 重置密码**
- 控制台 → 轻量应用服务器 → 实例详情 → 重置密码
- 设置 16 位以上强密码（数字+大小写字母+符号）

**② 配置防火墙**
- 控制台 → 防火墙 → 添加规则：

| 来源 | 端口 | 协议 | 说明 |
|------|:---:|------|------|
| 0.0.0.0/0 | 80 | TCP | HTTP |
| 0.0.0.0/0 | 443 | TCP | HTTPS |
| 0.0.0.0/0 | 22 | TCP | SSH（管理用） |

- **绝对不要**开放 3306、6379、3000 端口

**③ 记录公网 IP**
- 控制台首页就能看到，记下来

## 第三步：购买域名 + DNS 解析

### 3.1 买域名

1. 打开 [dnspod.cloud.tencent.com](https://dnspod.cloud.tencent.com)
2. 搜索你想要的域名，例如 `3c-lite.com`、`3czhineng.cn`
3. 选一个未被注册的：
   - `.com` ≈ **60 元/年**（首年常有优惠，30 元左右）
   - `.cn` ≈ **30 元/年**
4. 加入购物车 → 结算 → 微信/支付宝
5. 买后需**实名认证**：上传身份证 → 1 个工作日内通过

### 3.2 DNS 解析

1. 腾讯云控制台 → DNS 解析 DNSPod → 找到你的域名
2. 点击"添加记录"，添加两条：

| 主机记录 | 记录类型 | 记录值 | TTL |
|----------|:------:|------|:---:|
| `@` | A | 你的服务器公网 IP | 600 |
| `www` | A | 你的服务器公网 IP | 600 |

3. 保存，等 5-10 分钟生效
4. 验证：本地 PowerShell 里 `ping 你的域名`，应返回你的服务器 IP

### 3.3 域名备案（如果服务器在大陆）

- **必须备案**：域名 + 大陆服务器
- **不用备案**：买的香港/新加坡节点的服务器
- 备案流程（在腾讯云控制台提交）：
  1. 提交主体信息和网站信息
  2. 腾讯云初审：1 个工作日
  3. 管局审核：15-20 个工作日（期间网站不能访问）
- **建议**：如果赶时间，先买香港节点服务器上线 → 备案通过后再迁移到大陆节点

## 第四步：申请 SSL 证书（免费）

1. 访问 [console.cloud.tencent.com/ssl](https://console.cloud.tencent.com/ssl)
2. 点击"申请免费证书" → 选择 **TrustAsia 免费 DV 证书**（1 年有效）
3. 绑定域名：填写 `你的域名.com`
4. 验证方式选 **DNS 验证**（推荐，最快）
5. 腾讯云会自动添加 DNS 验证记录，等待 5-10 分钟
6. 签发后 → 下载 → 选择 **Nginx 格式**
7. 得到两个文件：
   - `xxx_bundle.crt` — 证书文件
   - `xxx.key` — 私钥文件
8. **妥善保存**这两个文件，马上要用

> 证书每年过期，到期前腾讯云会发短信提醒，届时重新申请替换即可。

## 第七步：配置环境变量（关键步骤）

### 7.1 生成 4 组密钥

```bash
cd /opt/3c-retail/backend

echo "JWT_SECRET: $(openssl rand -hex 32)"
echo "DB_ROOT_PASS: $(openssl rand -base64 24)"
echo "DB_USER_PASS: $(openssl rand -base64 24)"
echo "REDIS_PASS: $(openssl rand -base64 24)"
```

**把这 4 组输出保存到安全的地方**，后续运维需要用到。

### 7.2 创建 .env 文件

```bash
cp .env.example .env
chmod 600 .env
vim .env
```

vim 操作提示：按 `i` 进入编辑模式 → 编辑完成后按 `Esc` → 输入 `:wq` 回车保存。

**把下面内容逐项替换后填入**（`<角括号>` 部分替换为你的真实值）：

```ini
# ---- 应用 ----
APP_NAME=3c-retail-api
APP_PORT=3000
APP_ENV=production
CORS_ORIGIN=https://你的域名.com

# ---- 数据库 ----
DATABASE_URL="mysql://3capp:替换为DB_USER_PASS@mysql:3306/3c_retail?charset=utf8mb4&connection_limit=20"

# ---- Redis ----
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=替换为REDIS_PASS
REDIS_DB=0

# ---- JWT ----
JWT_SECRET=替换为JWT_SECRET
JWT_EXPIRES_IN=8h
JWT_REFRESH_EXPIRES_IN=7d

# ---- 日志 ----
LOG_LEVEL=info

# ---- Swagger ----
SWAGGER_ENABLED=false

# ---- Dify AI（先填占位值，等第九步配置完 Dify 再回来改）----
DIFY_BASE_URL=https://dify.你的域名.com/v1
DIFY_API_KEY=app-待填写

# ---- Docker Compose ----
MYSQL_ROOT_PASSWORD=替换为DB_ROOT_PASS
MYSQL_DATABASE=3c_retail
MYSQL_USER=3capp
MYSQL_PASSWORD=替换为DB_USER_PASS
```

### 7.3 验证

```bash
# 确保没有遗漏的占位值
grep -E "change-me|your-domain|待填写|替换为" .env && echo "还有未修改的占位值" || echo "配置检查通过"
```

## 第九步：部署 Dify AI 平台

Dify 是 AI 智能体的"发动机"，你小程序的 AI 对话功能依赖它。

### 9.1 方案选择

| 方案 | 费用 | 适合 |
|------|:----:|------|
| **自部署 Dify（推荐）** | 免费，占服务器 2G 内存 | 有一定运维能力 |
| Dify Cloud（SaaS） | Sandbox 免费/Professional $59/月 | 不想管运维 |

### 9.2 自部署 Dify（推荐）

```bash
# 装 git（如果还没装）
apt-get install -y git

# 拉取 Dify
cd /opt
git clone https://github.com/langgenius/dify.git
cd dify/docker
cp .env.example .env

# 生成 Dify 的密钥
echo "DIFY_SECRET: $(openssl rand -hex 32)"

# 编辑 Dify 的 .env，改 3 项：
vim .env
#   SECRET_KEY=上面生成的密钥
#   INIT_PASSWORD=自己设一个管理员密码
#   NGINX_PORT=8080

# 启动 Dify
docker compose up -d
```

Dify 启动后访问 `http://你的域名:8080` → 用刚才设的密码登录。

> 如果服务器是 2核4G，同时跑主应用和 Dify 可能会内存不足。要么买 4核8G（158元/月），要么另买一台 2核2G（68元/月）专门跑 Dify。

### 9.3 配置 AI 大模型

在 Dify 后台 → 设置 → 模型供应商 → 添加：

| 模型 | API 获取地址 | 费用 | 推荐 |
|------|-------------|:----:|:----:|
| **DeepSeek** | [platform.deepseek.com](https://platform.deepseek.com) | ¥1/百万 token ≈ 对话 ¥0.001/次 | ⭐首选 |
| 通义千问 | [dashscope.aliyun.com](https://dashscope.aliyun.com) | 有免费额度 | 备选 |
| OpenAI | [platform.openai.com](https://platform.openai.com) | $0.01-0.03/次对话 | 贵但效果好 |

注册对应平台 → 创建 API Key → 填入 Dify。

### 9.4 创建 Dify 应用 + 获取 API Key

1. Dify 后台 → 创建应用 → 选择"聊天助手"
2. （可选）创建知识库，上传你的产品信息、退换货政策等文档，关联到应用
3. 配置 Prompt：给 AI 设定 3C 数码销售助手的角色
4. 发布应用 → 获取 API 密钥（`app-xxx` 格式）

### 9.5 回填主应用的 .env

把 Dify API 密钥填回主项目配置：

```bash
cd /opt/3c-retail/backend
vim .env
```

修改这两行：

```ini
DIFY_BASE_URL=https://dify.你的域名.com/v1    # 自部署填这个
# DIFY_BASE_URL=https://api.dify.ai/v1         # 如果用 Dify Cloud 填这个
DIFY_API_KEY=app-你的Dify应用密钥
```

### 13.3 配置 COS 远程备份（强烈建议）

万一服务器磁盘坏了，COS 备份能救命。

```bash
# 安装 COS CLI
pip3 install coscmd

# 腾讯云控制台 → 对象存储 COS → 创建存储桶
#   - 名称随意，地域选和服务器同地域
#   - 访问权限：私有读写

# 控制台 → 访问密钥 → API密钥管理 → 获取 SecretId / SecretKey

coscmd config -a 你的SecretId -s 你的SecretKey -b 你的存储桶名 -r ap-guangzhou
echo "test" > /tmp/test.txt
coscmd upload /tmp/test.txt test.txt  # 测试
coscmd list
```

COS 费用 ≈ 每月 **2-5 元**。

## 第十四步：小程序端配置 + 发布

### 14.1 修改 API 地址

在本地开发者工具中打开 `miniapp/` 目录，修改环境配置，把 API 地址写死为：

```javascript
apiBase: 'https://你的域名.com/api'
```

（不要用 localhost，微信小程序要求必须是已备案的 HTTPS 域名）

### 14.2 配置服务器域名白名单

1. 登录 [mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 开发 → 开发管理 → 开发设置 → 服务器域名
3. 点击修改：
   - **request 合法域名**：添加 `https://你的域名.com`
4. 保存（**每月只能改 5 次**，确认无误再点）

### 14.3 上传代码 + 提交审核

1. 微信开发者工具打开项目 → 编译确认无误
2. 右上角"上传" → 版本号填 `1.0.0` → 备注"首次发布"
3. 进入小程序管理后台 → 版本管理
4. 开发版本中找到刚上传的 → 点击"提交审核"
5. 填写审核信息：
   - **服务类目**：按经营资质选择（如：商家自营 → 数码产品）
   - **测试账号**：给审核人员一个能登录的账号
6. 提交 → 等待审核 **1-7 个工作日**

### 14.4 审核通过 → 发布

审核通过后，在小程序后台点击"发布"，用户即可搜索到你的小程序。

## 附录 C：故障速查

| 现象 | 最可能原因 | 怎么做 |
|------|-----------|--------|
| 服务器连不上 | 密码错/防火墙没开22 | 控制台重置密码；检查防火墙规则 |
| 域名访问不通 | DNS 未生效 | `ping 域名` 看是否解析到你的 IP |
| HTTPS 不安全 | 证书没放对目录 | 检查 `nginx/ssl/` 下有无 `fullchain.pem` |
| 健康检查失败 | 数据库/Redis 连不上 | `docker compose logs app` 看错误 |
| 小程序审核被拒 | 类目不符/功能不全 | 看拒回原因，修改后重新提交 |
| 内存不够 | 服务太多 | 升级服务器配置或减少 Dify 容器内存限制 |
